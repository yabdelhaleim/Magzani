# 📋 توثيق نظام إدارة المخزون والمبيعات (Magzani ERP)

## نظرة عامة على النظام

نظام **Magzani ERP** هو نظام متكامل لإدارة المخزون والمبيعات مبني بلغة PHP بإطار عمل Laravel. يغطي النظام جميع جوانب العمل من المخازن إلى المحاسبة والتقارير.

---

## ١. إدارة المنتجات (Product Management)

### 1.1 إنشاء وتحديث وحذف المنتجات
- إنشاء منتجات جديدة ببيانات كاملة (الاسم، الكود، الباركود، الفئة، الوصف)
- تحديث بيانات المنتجات في أي وقت
- حذف المنتجات غير المستخدمة مع الحفاظ على السجلات التاريخية

### 1.2 دعم وحدات القياس المتعددة
- دعم **وحدة أساسية** لكل منتج (كيلوغرام، قطعة، صندوق، إلخ)
- إضافة **وحدات بيع** متعددة لكل منتج (كيلو، قطعة، صنف، كرتونة)
- تحويل تلقائي بين الوحدات (مثال: 1 كرتونة = 12 قطعة)

### 1.3 إدارة الباركود والرمز
- إدارة **الباركود** (Barcode) لكل منتج
- إدارة **الرمز التعريفي** (SKU) الداخلي
- البحث بالمنتج عبر الباركود أو الرمز

### 1.4 تصنيفات المنتجات
- إنشاء فئات رئيسية وفرعية
- تصنيف المنتجات حسب الفئة
- تصفية المنتجات حسب الفئة

### 1.5 تسعير المنتجات
- تحديد **السعر الأساسي** للوحدة الأساسية
- تسجيل **سجل تغيير الأسعار** للرجوع إليه
- دعم **التسعير المقترح** بالذكاء الاصطناعي بناءً على منتجات مماثلة

### 1.6 تنبيهات المخزون المنخفض
- تحديد **الحد الأدنى** للمخزون لكل منتج في كل مخزن
- تنبيه تلقائي ketika المخزون يقل عن الحد الأدنى
- عرض حالة المخزون (متوفر، محجوز، منخفض)

### 1.7 تحديث الأسعار الجماعي
- اختيار عدة منتجات دفعة واحدة
- تطبيق نسبة زيادة أو نقصان
- أو تحديد سعر ثابت للجميع

### 1.8 حالة المخزون الحية
- عرض المخزون المتوفر فعلياً
- عرض المخزون المحجوز للطلبات
- عرض إجمالي المخزون

## ٢. إدارة المخازن (Warehouse Management)

### 2.1 إنشاء وتحديث وحذف المخازن
- إضافة مخازن جديدة (الاسم، الموقع، المسؤول)
- تحديث بيانات المخزن
- حذف المخازن مع نقل المخزون أولاً

### 2.2 التحويلات بين المخازن
- إنشاء **أوامر تحويل** بين المخازن
- تحديد المنتج والكمية المطلوبة للتحويل
- تتبع **حالة التحويل**:
  - مسودة (draft)
  - معلقة (pending)
  - في الطريق (in_transit)
  - مستلمة (received)
  - ملغاة (cancelled)
  - معكوسة (reversed)

### 2.3 عكس التحويلات
- عكس التحويلات المكتملة
- إعادة المخزون للمخزن الأصلي تلقائياً

### 2.4 المخزون المتعدد المخازن
- عرض المخزون لكل مخزن على حدة
- إجمالي المخزون عبر جميع المخازن
- بحث المنتجات داخل مخزن معين

### 2.5 تقارير المخازن
- تقارير مستويات المخزون
- تقارير المخزون المنخفض
- تقارير قيمة المخزون

### 2.6 مدير المخزن
- تعيين مسؤول لكل مخزن
- صلاحيات محددة لمسؤول المخزن

---

## ٣. إدارة فواتير المبيعات (Sales Invoices)

### 3.1 إنشاء فاتورة مبيعات
- اختيار العميل من قائمة العملاء
- إضافة منتجات متعددة للفاتورة
- تحديد الوحدة لكل منتج (قطعة، كيلو، صندوق)
- إدخال الكمية والسعر
- إضافة الخصومات

### 3.2 الخصومات
- **خصم على الصنف**: نسبة أو مبلغ ثابت
- **خصم على الفاتورة**: نسبة أو مبلغ ثابت
- تطبيق الضرائب بعد الخصومات

### 3.3 حساب الضرائب
- إضافة نسبة ضريبة تلقائياً
- احتساب الضريبة لكل صنف
- إجمالي الضريبة على الفاتورة

### 3.4 الدفعات الجزئية
- تسجيل دفعة أولى جزئية
- تسجيل دفعات متعددة
- تتبع المبلغ المتبقي

### 3.5 حد الائتمان
- تحديد حد ائتمان لكل عميل
- منع تجاوز الحد المسموح
- تنبيه عند اقتراب من الحد

### 3.6 حالات الفاتورة
- مسودة (draft)
- مؤكدة (confirmed)
- ملغاة (cancelled)

### 3.7 حالات الدفع
- مدفوعة بالكامل (paid)
- دفعات جزئية (partial)
- غير مدفوعة (unpaid)

### 3.8 طباعة الفاتورة
- تصميم قابل للطباعة
- بيانات الشركة الكاملة
- تفاصيل الأصناف والأسعار

### 3.9 إلغاء الفاتورة
- إلغاء الفاتورة مع_return المخزون للمخزن
- تسجيل سبب الإلغاء

## ١٥. قاعدة البيانات (Database Models)

### النماذج (36 نموذج):

**المستخدمون والأدوار:**
- User (المستخدم)
- Role (الدور)
- Permission (الصلاحية)

**المنتجات:**
- Product (المنتج)
- Category (الفئة)
- Unit (الوحدة)

**المخازن:**
- Warehouse (المخزن)
- ProductWarehouse (مخزون المخزن)

**فواتير المبيعات:**
- SalesInvoice (فاتورة مبيعات)
- SalesInvoiceItem (صنف الفاتورة)

**فواتير المشتريات:**
- PurchaseInvoice (فاتورة مشتريات)
- PurchaseInvoiceItem (صنف الفاتورة)

**المرتجعات:**
- SalesReturn (مرتجع مبيعات)
- SalesReturnItem (صنف المرتجع)
- PurchaseReturn (مرتجع مشتريات)
- PurchaseReturnItem (صنف المرتجع)

**التحويلات:**
- WarehouseTransfer (تحويل مخزن)
- WarehouseTransferItem (صنف التحويل)

**الحسابات:**
- Customer (العميل)
- Supplier (المورد)

**الجرد:**
- StockCount (جلسة الجرد)
- StockCountItem (صنف الجرد)

**المحاسبة:**
- InventoryMovement (حركة مخزون)
- StockAdjustment (تسوية مخزون)
- Payment (دفعة)
- CashTransaction (معاملة نقدية)
- Expense (مصروف)
- ExpenseCategory (فئة المصروف)

**الوحدات والأسعار:**
- ProductBaseUnit (الوحدة الأساسية)
- ProductSellingUnit (وحدة البيع)
- ProductBasePricing (تسعير أساسي)
- ProductPriceHistory (سعر التاريخ)
- PriceChangeHistory (تغيير السعر)

**المالية:**
- SupplierPayment (دفعة للمورد)
- SupplierProductPrice (سعر المنتج للمورد)

**النظام:**
- ActivityLog (سجل النشاطات)
- SystemSetting (إعدادات النظام)
- Company (الشركة)
